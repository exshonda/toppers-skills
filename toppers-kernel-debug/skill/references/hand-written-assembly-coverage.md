# 手書きアセンブリ（`.S`）のカバレッジを実行トレースで測る（実装非依存）

ソースカバレッジツール（`gcov` 等）は**手書きアセンブリや最適化済みコード**には効きにくい。
依存部（起動・コンテキスト切替・割込み/例外入口）は手書きアセンブリで書かれることが多く、
その**命令(C0)・分岐(C1)網羅**を別手段で測る必要がある。本書は「**実行トレース＋デバッグ行情報の
突合**」による計測の原理と、その**ツールチェーン一般の落とし穴**をまとめる（実装非依存）。

> - 残存分岐の分類・台帳化、SMPで分岐が低く出る理由は [`multiprocessor-testing.md`](multiprocessor-testing.md)。
> - 踏めない経路への到達手段（フック流用・アイドル文脈）は [`reaching-unreachable-contexts.md`](reaching-unreachable-contexts.md)。
> - カーネル/リンカ無改変の禁則は別skill **`toppers-kernel-dev`** の `references/inviolable-rules.md`。
> - **具体的なエミュレータ・逆アセンブラ・スクリプト・フラグ名はリポジトリ/ツール固有**。本書は原理のみ。

---

## 1. 原理：逆アセンブル（母数）× 実行トレース（実行）× 行情報（対応）

ソースが C か手書きかに依らず、**最終バイナリにデバッグ行情報があれば**測れる。

- **母数（C0）**＝逆アセンブルで全命令アドレスを列挙。
- **実行（C0）**＝エミュレータ/トレーサで**実行した命令（または基本ブロック）列**を収集し、母数と突合。
- **対応**＝アドレス→ソース行マップ（デバッグ行情報）で命令網羅を行網羅に変換。
- **分岐（C1）**＝条件分岐ごとに taken先/fall先の2エッジを母数とし、トレースの
  「前ブロック末尾命令 → 次ブロック先頭」遷移と突合して taken/fall を判定。
- **SMPはPE（コア）別にトレースをタグ付け**して集計する。

*〔分類: カバレッジ/手書きコード ｜ プロセッサ: 非依存（ISA／デバッグ情報／実行トレーサ依存）｜ 出所: TTSP3(依存部.S計測)〕*

---

## 2. 落とし穴：終了しないテストでトレースが爆発する

終了マーカーを出さず**アイドルで回り続ける**テスト（時間管理系など）は、実行トレースが
無限に増大し**ディスクを食い潰す**。

- **サイズ上限つきのストリームフィルタで受け、上限超は読み捨てる（drain）**＋**タイムアウト**で抑える。
- 注意：上限到達で**読み口を閉じる**と、トレース生成側が書込みでブロックし結局タイムアウトまで
  待たされる。**読み続けて捨てる（drain）が正解**。
- 捕捉は**先頭プレフィックス＝実際に通ったエッジのみ**になるので、C1 を過大報告しない（健全）。

*〔分類: カバレッジ/トレース運用 ｜ プロセッサ: 非依存（実行トレーサ依存）｜ 出所: TTSP3〕*

---

## 3. 落とし穴：デバッグ情報のファイル名が解決できない（版差）

新しい既定のデバッグ情報版では、**手書きアセンブリのファイル名が別の文字列セクション**に格納される
ことがある。**リンカスクリプトの「保持するデバッグセクション」列挙にそのセクションが漏れている**と、
最終バイナリから**脱落**し、アドレス→ファイル名の解決が失敗する（行番号は復元できるため、
C ソースは別経路で解決できて**気付きにくい**）。

- **回避は、カーネルもリンカスクリプトも触らず、コンパイラのデバッグ情報版指定だけで
  旧版に落とす**（ファイル名が従来の文字列セクションに入る版を選ぶ）。＝無改変を維持。
- 版指定は**環境変数経由**で渡す。フラグ変数を**コマンドラインで丸ごと代入**すると、makefile が
  足していたアーキ指定等が消えてアセンブルエラーになる（**追加**であって**置換**ではない形で渡す）。

*〔分類: カバレッジ/ビルド ｜ プロセッサ: 非依存（DWARF等のデバッグ情報・リンカ依存）｜ 出所: TTSP3〕*

- **同じ「新デバッグ情報版のセクション漏れ」は、カバレッジだけでなく OS Awareness 等の
  デバッガによるシンボル読取りそのものを壊す**。リンカスクリプトの保持リスト（とくに
  末尾の `/DISCARD/` 一括破棄）に新版のセクション群が無いと、それらが最終バイナリから
  落ち、デバッガが「必要なセクションが無い」でシンボル解決に失敗する。**リンカスクリプトを
  自分で生成・所有している場合**は、旧版へ落とす代わりに**保持リストへ新版のセクションを
  追加する**選択肢もある（生成テンプレート側に一度入れれば全ターゲットに効く）。
  <br>*〔分類: 観測/デバッガ ｜ プロセッサ: 非依存（新デバッグ情報版のセクション保持漏れ） ｜ 出所: stm32mp2_baremetal〕*

---

## 4. 落とし穴：別リンクのバイナリは命令アドレスが非互換

各テストが**別々にリンクされた別バイナリ**だと、同じソースでも命令アドレスが食い違い、
**命令アドレスの単純な union（合算）は壊れる**。

- 合算は**(ファイル, 行 [, 分岐]) の空間で行う**（アドレス空間でなく、ソース位置空間で重ねる）。
- 各テストのアドレス→行解決は**テストごとに異なる相対パス**を返しがち。**絶対パスへ正規化してから**
  合算する（でないと同一ソースが別ファイル扱いになり連結が壊れる）。

*〔分類: カバレッジ/合算 ｜ プロセッサ: 非依存 ｜ 出所: TTSP3〕*

---

## 5. union 安全則：合算前にクリーン再ビルド

複数テストのカバレッジを合算する前は**クリーン再ビルド**する。古い計装/メタ情報が混ざると
**分母が揺れ**、単なる再コンパイル差を「カバレッジ向上」と誤認する。これは手書きアセンブリ計測でも
ソース計測でも同じ規律。

*〔分類: カバレッジ/合算 ｜ プロセッサ: 非依存 ｜ 出所: TTSP3〕*

## まとめ

| 課題 | 対処 |
|---|---|
| gcov が効かない手書き/最適化コード | 逆アセンブル×実行トレース×デバッグ行情報の突合（C0/C1）。SMPはPE別タグ |
| 終了しないテストでトレース爆発 | サイズ上限フィルタで drain ＋ タイムアウト（読み口は閉じない） |
| ファイル名が解決できない | デバッグ情報版を旧版に（無改変・環境変数で**追加**指定） |
| 別リンクでアドレス非互換 | (ファイル,行,分岐)空間で合算・パスは絶対正規化 |
| 合算で分母が揺れる | 合算前にクリーン再ビルド |

> 具体的なエミュレータ／逆アセンブラ／フィルタ実装／フラグ・スクリプト名は**リポジトリ/ツール固有**。
> 本書の原理を、各リポジトリの `AGENTS.md`／`docs/`／専用skillの手順に結びつけて使う。
